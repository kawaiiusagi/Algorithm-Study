# 2476 주사위 게임
[백준 2476 주사위 게임]

### 문제 요약
세 개의 주사위를 던졌을 때 나오는 눈의 조합에 따라 상금을 계산하고, **여러 사람 중 가장 큰 상금을 받은 금액을 출력하는 문제**이다.

### 코드
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main()
{
	int n;
	cin >> n;

	vector<int> total;

	for (int i = 0; i < n; i++) {
		vector<int> temp;

		for (int j = 0; j < 3; j++) {
			int x;
			cin >> x;
			temp.push_back(x);
		}
		sort(temp.begin(), temp.end());

		if (temp[0] == temp[1] && temp[1] == temp[2]) {
			int n = 10000 + (temp[0] * 1000);
			total.push_back(n);
		}
		else if (temp[0] == temp[1]) {
			int n = 1000 + (temp[0] * 100);
			total.push_back(n);
		}
		else if (temp[1] == temp[2]) {
			int n = 1000 + (temp[1] * 100);
			total.push_back(n);
		}
		else {
			int n = temp[2] * 100;
			total.push_back(n);
		}
	}
	sort(total.begin(), total.end());
	
	cout << total[n - 1];

	return 0;
}
```
---
### 풀이 과정 및 느낀 점
- 이번 문제에서는 조건 분기가 많아 코드가 불필요하게 길어지는 느낌을 받아, 최대한 간결하게 줄이려는 방향으로 구현하려고 했다.
- 하지만 결국 세 가지 경우(모두 같음, 두 개만 같음, 모두 다름)를 나누는 큰 틀은 바뀌지 않아 구조 자체는 비슷하게 유지되었다.
- 특히 두 개의 값이 같을 때, 어떤 값이 같은지 추출하는 과정이 다소 번거롭게 느껴졌고, 이를 더 간단하게 구현할 수 있는 방법을 고민해보고 싶다는 생각이 들었다.
- 조건을 명확히 정리한 뒤 구현하면 생각보다 복잡하지 않은 문제였지만, 조건 처리 방식에 따라 코드의 길이와 가독성이 크게 달라진다는 점을 느꼈다.

### 내가 느낀 중요한 점
- 조건 분기를 구조적으로 정리하는 알고리즘 구현 능력
