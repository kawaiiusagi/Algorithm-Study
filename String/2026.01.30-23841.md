# 23841 데칼코마니
[백준 23841번 데칼코마니](https://www.acmicpc.net/problem/23841)

### 문제 요약
주어진 문자 배열을 가로로 절반씩 나누어, 왼쪽 문자와 오른쪽 문자를 비교하며 물음표(`?`)를 적절한 문자로 채워 완성된 문자열을 출력하는 문제이다.

### 코드
```cpp
#include <iostream>
#include <vector>
#include <string>

using namespace std;

int main() {
    int N, M;
    cin >> N >> M;

    vector<string> a(N);
    for (int i = 0; i < N; i++) {
        cin >> a[i];
    }

    for (int i = 0; i < N; i++) {
        for (int j = 0; j < M / 2; j++) {
            int k = M - 1 - j;

            if (a[i][j] == '.' && a[i][k] != '.') {
                a[i][j] = a[i][k];
            } 
            else if (a[i][j] != '.' && a[i][k] == '.') {
                a[i][k] = a[i][j];
            }
        }
    }

    for (int i = 0; i < N; i++) {
        cout << a[i] << '\n';
    }

    return 0;
}

```

### 풀이 과정 및 느낀 점
- 처음에는 문제를 단순히 문자열을 채우는 문제라고 생각했지만, 실제로는 **문자열을 좌우 대칭 구조로 바라보는 사고**가 필요했다.
- 각 행마다 왼쪽과 오른쪽 문자를 동시에 비교해야 했고, 한쪽만 문자가 존재하고 다른 쪽이 `?`인 경우를 구분해 처리해야 했다.
- 특히 양쪽이 모두 `?`인 경우와, 한쪽만 `?`인 경우를 나누어 생각하지 않으면 조건 분기가 복잡해졌다.
- 구현 과정에서 인덱스 계산을 잘못해 여러 번 오류가 발생했는데, 문자열 길이를 기준으로 **좌우 인덱스를 정확히 대응시키는 것이 핵심**이라는 점을 다시 느꼈다.
- 이번 문제를 통해 문자열 문제에서도 **대칭 구조와 인덱스 설계가 얼마나 중요한지**를 배울 수 있었다.

---

### 이 문제에서 중요한 점
- 문자열을 **좌우 대칭 구조**로 해석하는 사고
- 인덱스를 이용한 정확한 위치 접근
- 조건에 따른 문자 대체 로직 구현
