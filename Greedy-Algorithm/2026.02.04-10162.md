# 5585 전자레인지
[백준 10612 전자레인]([https://www.acmicpc.net/problem/5585](https://www.acmicpc.net/problem/10162))

### 문제 요약
주어진 시간을 300초, 60초, 10초 버튼으로 최소 횟수로 맞추거나, 정확히 만들 수 없으면 -1을 출력하는 그리디 알고리즘 문제이다.

### 코드
```cpp
#include <iostream>
#include <vector>
#include <string>

using namespace std;

int main() {
	int n;
	cin >> n;

	int buttons[4] = {300, 60, 10};
	vector<int> res;
	
	int i = 0;
	string x = to_string(n);

	for (int i = 0; i < 3; i++) {
		int ans = 0;
		ans += n / buttons[i];
		res.push_back(ans);

		n %= buttons[i];
	}

	if (n != 0) {
		cout << "-1";
	}
	else {
		for (int x : res) {
			cout << x << " ";
		}
	}
	return 0;
}
```
---
- 이번 문제를 통해 그리디 알고리즘을 처음 접했는데, 이전에 배웠던 자릿수 관련 알고리즘과 유사하면서도 **훨씬 간결하고 빠르게 문제를 해결**할 수 있는 방식이라는 점이 인상적이었다.
- 가장 큰 단위부터 먼저 처리하는 방식으로 불필요한 계산을 줄일 수 있었고, 그리디 알고리즘의 핵심적인 사고 흐름을 이해할 수 있었다.
- 구현 과정에서 **`to_string`을 사용해 자료형을 `string`으로 변환하는 방법**을 새롭게 알게 되었고, 자료형 변환 함수들의 이름이 직관적이지 않아 헷갈릴 수 있다는 점도 함께 느꼈다.
- 처음에는 시간의 단위가 10초 단위이므로 입력값이 30의 배수가 아닐 경우 바로 `-1`을 출력하려고 했으나, 모든 연산을 수행한 뒤에도 값이 남아 있는지를 조건으로 판단하면 훨씬 간단하게 구현할 수 있다는 피드백을 통해 로직을 개선할 수 있었다.

### 내가 생각하는 이 문제에서 중요한 점

- **조건 분기**를 줄여 로직을 간결하게 만드는 구현 방식
