# 1920 수 찾기
[백준 1920번 수 찾기](https://www.acmicpc.net/problem/1920)

### 문제 요약
정수 N개가 주어질 때, 또 다른 정수들이 해당 집합에 존재하는지 이분 탐색을 이용해 빠르게 판별하는 문제이다.

### 코드
```cpp
#include <iostream>
#include <algorithm>

using namespace std;

int arr[100001], brr[100001];

int binarySearch(int ar[], int len, int target) {
	int low = 0;
	int high = len - 1;

	while (low <= high) {
		int mid = (low + high) / 2;
		if (ar[mid] == target) {
			return 1;
		}
		else if (ar[mid] < target) {
			low = mid + 1;
		}
		else {
			high = mid - 1;
		}
	}
	return 0;
}

int main()
{
	int n, m;
	cin >> n;
	
	for (int i = 0; i < n; i++) {
		cin >> arr[i];
	}
	sort(arr, arr+n);
	int len = n;

	cin >> m;

	for (int i = 0; i < m; i++) {
		cin >> brr[i];
	}
	for (int i = 0; i < m; i++) {
		int res = binarySearch(arr, len, brr[i]);

		if (res == 1) {
			cout << "1" << "\n";
		}
		else {
			cout << "0" << "\n";
		}
	}
	return 0;
}
```
---
### 풀이 과정 및 느낀 점

- 처음 작성한 코드는 논리적으로는 문제없이 작동했지만, 시간 초과가 발생하여 원인을 찾는 데 어려움을 겪었다.
- 단순 탐색 방식으로 구현하면 시간 복잡도가 커지기 때문에, 정렬 후 이분 탐색을 사용하는 방식으로 수정하였다.
- 이진 탐색의 개념 자체는 어렵지 않았지만, `low`, `high`, `mid`의 범위 설정과 반복 조건을 정확히 구현하는 과정에서 여러 오류를 경험하였다.
- 특히 알고리즘 문제에서는 정답 여부뿐만 아니라 **시간 복잡도까지 고려해야 한다는 점**을 다시 한 번 체감하였다.
- 앞으로는 구현 이전에 시간 복잡도를 먼저 계산해보고, 더 효율적인 방법이 있는지 고민하는 습관을 가져야겠다고 느꼈다.

### 중요한 포인트

- 정렬 + 이분 탐색을 통한 시간 복잡도 개선
- 구현 과정에서의 범위 설정과 반복 조건 관리
- 문제 풀이에서 시간 복잡도를 항상 고려하는 사고
