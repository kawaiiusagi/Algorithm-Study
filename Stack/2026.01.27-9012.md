# 9012 괄호
[백준 9012번 괄호](https://www.acmicpc.net/problem/9012)

### 문제 요약
괄호 문자열이 주어질 때, 열린 괄호와 닫힌 괄호가 올바르게 짝을 이루는지 판단하여 올바르면 YES, 아니면 NO를 출력하는 문제이다.

### 코드
```cpp
#include <iostream>
#include <stack>
#include <string>

using namespace std;

int main()
{
	int t;
	cin >> t;
	cin.ignore();

	for (int i = 0; i < t; i++) {
		stack<char> p;
		string s;
		getline(cin, s);

		for (int j = 0; j < s.length(); j++) {
			if (s[j] == ')' && !p.empty() && p.top() == '(') {
					p.pop();
				}
			else {
				p.push(s[j]);
			}
		}

		if (p.empty()) {
			cout << "YES" << endl;
		}
		if (!p.empty()) {
			cout << "NO" << endl;
		}
	}

	return 0;
}
```
- `stack` 초급 문제로 분류되어 있지만, 실제로는 문제의 핵심을 파악하지 못하면 접근하기 쉽지 않은 문제라고 느꼈다. 처음에는 왜 이 문제에서 `stack`을 사용해야 하는지 감이 잘 오지 않아 **풀이 방향을 잡는 데 어려움**이 있었다.
- 구현 과정에서 **`if`문을 같은 선상에서 여러 개 사용**하다 보니 논리 오류가 발생하였고, 이를 `else if` 구조로 정리한 뒤 점차 **조건을 단순화**하여 하나의 명확한 조건문으로 개선해 나갔다.
- 괄호 한 쌍이 완성될 때마다 `stack`에서 `pop()`하는 방식으로 코드를 구성했는데, 이 과정에서 `'('`보다 **`')'`를 기준으로 사고했다면 더 간결한 코드**로 처음부터 구현할 수 있었을 것이라는 아쉬움이 남았다. 초기 코드에는 불필요한 조건들이 포함되어 있어 다소 복잡해졌다.
- 또한 `cin`과 `getline`을 함께 사용할 경우 입력 버퍼에 남아 있는 개행 문자로 인해 문제가 발생할 수 있으므로, **`cin.ignore()`를 반드시 사용**해야 한다는 점을 다시 한 번 확실히 인식하게 되었다.

### 이 문제에서 내가 생각하는 중요한 포인트
- 문제의 특성을 파악하고 적절한 자료구조(`stack`)를 선택하는 **알고리즘 구상 능력**
