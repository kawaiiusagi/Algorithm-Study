# 10799 쇠막대기
[백준 10799 쇠막대기](https://www.acmicpc.net/problem/10799)

### 문제 요약
레이저와 쇠막대기가 주어질 때, 레이저로 잘려 생기는 쇠막대기 조각의 총 개수를 스택을 이용해 계산하는 문제이다.

### 코드
```cpp
#include <iostream>
#include <stack>
#include <string>
using namespace std;

int main() {

    string s;
    cin >> s;

    stack<char> st;
    int ans = 0;

    for (int i = 0; i < s.length(); i++) {
        if (s[i] == '(') {
            st.push('(');
        }
        else {
            st.pop();

            if (s[i - 1] == '(') {
                ans += st.size();
            }
            else {
                ans += 1;
            }
        }
    }

    cout << ans;

    return 0;
}
```
- 지금까지 풀었던 스택 문제들 중에서 가장 난이도가 높다고 느꼈다. 쇠막대기와 레이저를 구분하기 위한 알고리즘을 구상하고 이를 코드로 구현하는 과정에서 많은 어려움이 있었다.
- 문제를 해결하면서 아직 스택을 유연하게 활용하지 못하고 있으며, 사고가 지나치게 한정적이라는 점을 스스로 인식하게 되었다.
- 특히 전체 구조를 한 번에 스택으로 해결하려는 사고에 갇혀 오히려 문제를 더 복잡하게 만들었던 것 같다.
- 이번 문제를 통해 스택 문제에서는 **판단과 상태 관리의 역할을 분리해야 한다**는 점을 깨달았다. 즉, 스택을 판단의 도구로 사용하기보다는, 이미 판단이 끝난 결과를 저장하는 공간으로 바라보는 사고가 필요하다고 느꼈다.
- 앞으로는 스택 문제를 풀 때, 무엇을 판단해야 하는지와 무엇을 기억해야 하는지를 명확히 구분하여 알고리즘을 설계하고 구현해야겠다고 생각했다.
- 또한 개인적으로 비슷한 자료구조인 `Queue`에 비해 `Stack`을 더 어렵게 느끼고 있어, 다양한 유형의 스택 문제를 추가로 풀며 사고를 확장할 필요성을 느꼈다.

### 이 문제에서 중요한 부분
- 스택의 상태 관리 방식에 대한 이해
- 문제 상황을 분해하여 단계적으로 해결하는 알고리즘 구상 및 구현 능력
