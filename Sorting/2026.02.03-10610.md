# 10610 30
[백준 10610번 30](https://www.acmicpc.net/problem/10610)

### 문제 요약
주어진 숫자의 각 자릿수를 재배열하여 30의 배수가 되는 가장 큰 수를 만들고, 만들 수 없으면 -1을 출력하는
문제이다.

### 코드
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

bool cmp(int x, int y) {
	return x > y;
}

int main()
{
	string n;
	int total = 0;
	cin >> n;

	vector<int> v;
	bool has_zero = false;

	for (int i = 0; i < n.size(); i++) {
		v.push_back(n[i] - '0'); // 이게중요
		total += v[i];

		if (v[i] == 0) {
			has_zero = true;
		}
	}


	if (total % 3 != 0 || !has_zero) {
		cout << "-1";
	}
	else {
		sort(v.begin(), v.end(), cmp);

		for (int x : v) {
			cout << x;
		}
	}

	return 0;
}
```
---

- 처음에는 로직을 올바르게 구현했다고 생각했지만, 입력으로 주어지는 숫자의 자릿수가 매우 커 `int`형으로 입력받을 경우 오버플로우가 발생해 오답이 나왔었다. 이를 통해 입력 조건을 먼저 확인하고 자료형을 선택하는 것이 중요하다는 점을 다시 한 번 느꼈다.
- 조건 판단 과정에서 `bool`형 자료형을 더 적극적으로 활용했다면 코드가 훨씬 간결해질 수 있었을 것 같아, 논리 판단을 효율적으로 표현하는 연습이 필요하다고 생각했다.
- 이번 문제를 통해 `n[i] - '0'` 표현을 확실히 이해하게 되었다. 문자열로 입력받은 숫자를 정수로 변환하는 과정에서 아스키 코드 개념을 활용하면 간단하게 처리할 수 있다는 점을 배울 수 있었다.

### 중요한 포인트
- 입력 조건을 정확히 해석하고, 그에 맞는 자료형을 선택하는 문제 해석 능력
